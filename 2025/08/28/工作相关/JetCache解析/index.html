1. JetCache 多级缓存架构

JetCache 是一个基于 Java 的缓存抽象框架，其核心目标是提供一个统一、易用且功能强大的缓存 API，让开发者可以轻松地集成和使用各种缓存提供商（如 Redis、Caffeine、Tair 等），而无需关心底层实现的细节。
JetCache 的核心特点：

  1. 统一的 API 和注解：通过简单的 @Cached 等注解即可实现方法缓存，编程方式也保持一致。
  2. 多级缓存支持：可以轻松构建“本地缓存（如 Caffeine） + 分布式缓存（如 Redis）”的多级缓存架构，自动处理数据传播和一致性。
  3. 自动缓存失效：支持通过 @CacheUpdate 和 @CacheInvalidate 注解自动更新或删除缓存。
  4. 丰富的配置选项：支持细粒度的 TTL（生存时间）、缓存穿透/击穿保护、统计监控等。
  5. Spring Boot 无缝集成：通过 jetcache-starter-redis 等组件，可以几乎零配置地集成到 Spring Boot 项目中。
  6. 序列化：JetCache 默认使用 Kryo 进行序列化，相比 Java 原生序列化或 Jackson，它的速度更快，产生的字节数组更小，非常适合缓存场景。也支持自定义为 JSON 等格式。

1.1 缓存类型配置

关键配置说明：

  • CacheType.BOTH: 启用多级缓存，同时使用本地缓存和远程缓存
  • syncLocal(true): 启用本地缓存同步，确保本地缓存与远程缓存的一致性
  • expire(Duration.ofDays(20)): 设置缓存过期时间，保证数据的时效性

1.2 多级缓存工作原理

缓存层级：

  1. L1缓存（本地缓存）: 使用Caffeine实现，提供最快的访问速度
  2. L2缓存（远程缓存）: 使用Redis实现，提供分布式共享能力

2. 会话持久性保障机制

会话持久化策略：

  • 命名空间隔离: 使用key前缀区分会话数据
  • 过期时间管理: 设置合理的过期时间，避免内存泄漏
  • UUID模式识别: 自动识别会话ID并添加命名空间

2.2 会话同步机制

同步策略：

  • 时间间隔控制: 避免频繁同步，提高性能
  • 异步同步: 使用异步任务进行数据库同步，不阻塞主流程
  • 增量同步: 只同步变更的会话数据

3. 缓存一致性保障机制

3.1 本地缓存同步

同步机制：

  1. 写操作同步: 当数据写入时，同时更新本地缓存和远程缓存
  2. 读操作策略: 优先从本地缓存读取，本地缓存未命中时从远程缓存读取
  3. 失效同步: 远程缓存失效时，自动清除本地缓存

3.2 缓存失效策略

失效策略：

  • 精确失效: 根据具体键值进行失效
  • 模式失效: 支持通配符模式批量失效
  • 级联失效: 本地缓存和远程缓存同时失效

3.3 会话状态一致性

一致性保障：

  • 变更标记: 会话属性变更时自动标记
  • 选择性同步: 只同步业务相关的属性变更
  • 状态追踪: 实时追踪会话状态变化

4. 高可用性保障

4.1 缓存降级策略

降级机制：

  • Redis可用: 使用Redis作为远程缓存
  • Redis不可用: 自动降级到EhCache本地缓存
  • 配置切换: 通过配置动态切换缓存策略

4.2 会话恢复机制

恢复策略：

  • 数据库备份: 会话数据持久化到数据库
  • 自动恢复: 缓存失效时从数据库恢复会话
  • 状态同步: 确保会话状态的一致性

总结

JetCache在若依项目中通过以下机制保证会话持久性和缓存一致性：
会话持久性保障：

  1. 多级缓存架构: 本地缓存+远程缓存的双重保障
  2. 数据库持久化: 会话数据同步到数据库作为备份
  3. 异步同步机制: 避免同步操作影响性能

缓存一致性保障：

  1. 自动同步机制: 本地缓存与远程缓存自动同步
  2. 失效传播: 缓存失效时级联清除相关缓存
  3. 状态追踪: 实时追踪会话状态变化

高可用性保障：

  1. 降级策略: Redis不可用时自动降级到本地缓存
  2. 恢复机制: 缓存失效时从数据库恢复数据
  3. 监控告警: 实时监控缓存和会话状态
